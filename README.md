\# SQL Module Project – Trendora Shopping Dataset Analysis



\## Overview
![SQL Project Screenshot](images/mysql%20module%20project%20image.png)
This project is a MySQL-based analysis of the Trendora Shopping Dataset, completed as part of the Torilo Academy Data Analysis Program. It demonstrates core SQL querying skills, from basic filtering and sorting to more advanced data retrieval techniques used to answer real business questions about customer shopping behavior.



\## Objective

The goal of this project was to explore and analyze the Trendora dataset using SQL to uncover patterns in customer purchases, product performance, and sales trends — and to demonstrate proficiency in writing clear, efficient, and well-structured SQL queries.



\## Tools Used

\- \*\*MySQL\*\* – database engine and query execution

\- \*\*MySQL Workbench\*\* (or equivalent client) – for writing and running queries

The dataset used is `Dataset.csv`, located in the `Data/` folder. It contains transactional and product-level shopping data used as the basis for all queries in this project.

\## What the Queries Cover

The assessment consisted of 15 SQL queries, split across two core areas:

\- \*\*SELECT \& WHERE clauses\*\* – filtering and retrieving specific subsets of data based on given conditions

\- \*\*ORDER BY, LIMIT \& OFFSET\*\* – sorting results and controlling pagination of returned data



Together, these queries answer practical business questions such as identifying top products, filtering customer segments, and ranking sales performance.



\## How to Use This Project

1\. Import `Data/Dataset.csv` into a MySQL database/table.

2\. Open `analysis\_queries.sql` in MySQL Workbench (or your preferred SQL client).

3\. Run the queries in order to reproduce the analysis.

4\. Refer to `Analysis\_Report.pdf` for the full breakdown of each query's purpose and findings.
